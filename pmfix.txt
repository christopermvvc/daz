PM Storage Bug Quick Summary
The Problem:
Private messages are received by the bot but never make it to the database, even though the infrastructure exists to store them.
What's Happening:

When someone sends a PM to the bot, the WebSocket client receives it correctly
The parser creates a proper PrivateMessageEvent with all data (from_user, to_user, message, timestamp)
BUG: The Room Manager receives this event but only passes it as a generic RawEvent instead of populating the PrivateMessageData field
The SQL plugin receives the event but can't extract PM-specific fields because they're not populated
Result: PM gets stored in database but with empty username, message, and to_user fields
